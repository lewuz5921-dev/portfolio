01

项目背景与痛点

传统背单词 App（百词斩、墨墨等）依赖"图片联想+重复打卡"模式，用户容易陷入"背了忘、忘了背"的死循环。核心痛点有二：一是单词孤立记忆缺乏语境锚点；二是"已掌握"状态缺乏真实验证——用户在收集阶段点击"认识"或挑战阶段猜对，不等于真正记住。

产品目标

设计一套"语境优先+复习重验"的背词系统：先通过情境故事建立语义关联，再通过"温故知新"对已掌握单词进行随机抽查重验，答错的单词自动扣回生词池重新学习，形成"记忆-检测-修复"的完整闭环。

02

目标用户与场景

面向雅思备考学生（目标 7.5+），尤其是背词效率低、容易遗忘、对传统 App 感到疲惫的用户。

  • 场景1：每天通勤/睡前，用10分钟完成一轮"收集+阅读+挑战"的闭环背词
  • 场景2：周末复习时，通过"温故知新"对已掌握单词进行重验，答错的自动扣回重新学习
  • 场景3：考前突击，通过完形挑战快速检测词汇掌握情况，结合复习重验定位薄弱词汇

04

AI 工作流设计

产品的核心是设计一套多轮 Prompt 工程，让 Gemini 在背单词场景中扮演"语境生成器""题目设计师"和"复习出题人"三重角色。

05

产品方案与交互设计

产品以"四轮闭环"为核心：先收集、再阅读、后挑战、终复习，每轮10个单词，轻量化完成。

生词收集二选一模式（认识/不认识），快速筛选出10个真正不熟悉的单词，降低认知负担
情境阅读Gemini 实时生成包含10个生词的短文，带 TTS 朗读，用户可划词查义、查看翻译
完形挑战从短文中提取真实例句，挖空目标单词生成选择题，检测真实掌握程度
温故知新从已掌握词池中随机抽取10个单词进行完形重验，答错的单词自动扣回生词池，强制重新学习
进度统计顶部实时展示已掌握/不认识/积攒生词数，学习成果可视化

为什么用"二选一"而非"四选一"收集生词？

传统 App 用四选一让用户"猜词义"，但猜对不等于认识。二选一（认识/不认识）让用户诚实面对自己的掌握程度，避免虚假记忆干扰后续学习。

为什么每轮固定10个单词？

认知心理学研究表明，工作记忆容量约为 7±2 个组块。10个单词是一个既能覆盖一定量、又不会造成认知超载的 sweet spot。

为什么设计"惩罚扣回"机制？

传统 App 的"已掌握"状态一旦标记很少改变，用户可能高估自己的掌握程度。惩罚扣回通过真实的完形重验，把"看着认识"但"用时想不起来"的单词强制拉回学习队列，确保记忆质量而非记忆数量。

06

核心流程

用户旅程：选择词库 → 二选一收集10个生词 → Gemini 生成情境短文 → 阅读短文（朗读/查义/翻译）→ 完成10道完形挑战 → 答对单词进入已掌握池 → 通过"温故知新"对已掌握单词进行重验 → 答错扣回生词池 → 继续下一轮。

  • 收集闭环：展示单词 → 用户判断认识/不认识 → 积攒满10个生词 → 进入阅读环节
  • 阅读闭环：加载中（Gemini 生成短文）→ 展示原文 → 朗读/划词查义/查看翻译 → 进入挑战
  • 挑战闭环：展示例句挖空题 → 4选1作答 → 即时反馈对错 → 10题完成后展示统计 → 答对单词进入已掌握池
  • 复习闭环：欢迎页点击"温故知新" → 从已掌握池随机抽取10个单词 → 完形重验 → 答错扣回生词池 → 展示复习重验报告
07

AI 驱动与 Prompt 工程

  • 设计多轮对话式 prompt，让 Gemini 同时承担"作家+出题人+翻译+复习出题人"四重角色
  • 通过 prompt 约束生成质量：要求句意简单、画面感强、避免生僻搭配
  • 复习题目基于牛津释义与形似选项干扰机制，增加区分度和检测有效性
  • 利用 Gemini Canvas 的代码生成能力，将 prompt 设计直接转化为可交互前端原型
  • 加入 TTS Web Speech API 实现朗读功能，无需额外后端
08

产品价值与预期

  • 将孤立单词记忆转化为语境语义记忆，预期记忆留存率提升 30%+
  • AI 实时生成个性化内容，每轮短文与复习题目不重复，保持学习新鲜感
  • 四轮闭环设计（收集→阅读→挑战→复习）形成完整的学习输入-加工-输出-修复链条
  • "惩罚扣回"机制确保已掌握状态的真实性，避免虚假记忆
  • Gemini Canvas 实现零代码原型开发，验证"AI Agent 驱动产品"的可行性
09

复盘与思考

  • 思考：AI 生成内容的质量一致性是关键挑战，需要精细的 prompt 约束和边界 case 处理
  • 思考：教育产品的核心不是"技术炫技"，而是"学习效果可感知"——用户需要明确感受到"我记住了"
  • 思考："惩罚扣回"机制在心理上可能带来挫败感，需要通过文案和进度可视化来平衡（如"复习重验报告"替代"不及格"）
  • 若推进：加入学习数据追踪（遗忘曲线、易错词统计、复习频率），为个性化复习提供数据支撑
  • 成长：通过这个项目理解了"Prompt 工程即产品设计"——在大模型时代，prompt 设计能力等同于产品逻辑设计能力

1

欢迎页 · 词库选择与温故知新入口

页面作用

产品入口，展示核心价值主张，提供词库选择、开始背词和"温故知新"复习入口

设计目的

3秒内传递"语境记忆"定位，同时让已掌握单词的复习入口一目了然

解决的问题

解决用户"从哪开始"的迷茫，以及已学单词缺乏复习入口的问题

为什么这样设计

大标题直击痛点，"温故知新"按钮实时显示已掌握单词数，制造复习紧迫感

对应业务流程

进入→选择词库→开始背词 / 温故知新复习

产品思考

"温故知新"按钮上的数字徽章（如"12"）让用户直观看到已积累的词汇量，同时暗示"这些单词需要被复习"

2

测试页 · 生词二选一收集

页面作用

展示目标单词，用户选择"认识"或"不认识"，积攒10个生词

设计目的

快速、诚实地筛选出真正不熟悉的单词

解决的问题

解决传统四选一"猜对≠认识"的虚假记忆问题

为什么这样设计

卡片式布局聚焦单单词，音标+发音按钮辅助判断，底部进度条可视化积攒进度

对应业务流程

展示单词→判断认识/不认识→积攒满10个→进入阅读

产品思考

"积攒"概念比"背诵"更轻量，降低用户心理压力；进度条制造完成感

3

阅读页 · AI 情境短文

页面作用

Gemini 生成的包含10个生词的情境短文，支持朗读、划词查义、翻译

设计目的

让用户在读懂故事的过程中自然建立单词与语境的语义关联

解决的问题

解决孤立背单词缺乏语义锚点的问题

为什么这样设计

原文与翻译分层展示，划词选中弹出释义气泡，TTS 朗读帮助建立音-义-形联结

对应业务流程

阅读短文→划词查义→查看翻译→进入挑战

产品思考

"情境场景抗遗忘阅读"标签强化产品定位，翻译按钮默认隐藏避免依赖

4

挑战页 · 语境完形填空

页面作用

从短文中提取的真实例句挖空题，4选1检测掌握程度

设计目的

通过"输出"检测真实掌握情况，而非"输入"时的短期记忆

解决的问题

解决"看着认识、用不出来"的典型问题

为什么这样设计

题目来自真实短文语境（非编造例句），选项设计包含近义词干扰，增加区分度

对应业务流程

展示题目→选择答案→即时反馈→10题完成后展示结果

产品思考

即时反馈（正确/错误）强化记忆，结果页展示已掌握 vs 需重修的单词列表

5

温故知新 · 复习重验与惩罚扣回

页面作用

从已掌握词池中随机抽取10个单词进行完形重验，答错扣回生词池

设计目的

真实检测已掌握单词的记忆质量，把"假性掌握"的单词拉回学习队列

解决的问题

解决"标记已掌握后就再也不看，实际早已遗忘"的问题

为什么这样设计

复习题目基于牛津释义与形似词干扰设计，增加检测难度；结果页明确标注"复习露馅（扣回生词池）"让用户清晰看到哪些单词需要重新学习

对应业务流程

点击温故知新→随机抽取10个已掌握单词→完形重验→答错扣回→展示复习重验报告

产品思考

把"惩罚"包装为"重验"和"修复"，降低挫败感；用数据可视化（已掌握数变化）让用户感受到复习的价值
